## Bugs found only by running live (all fixed + regression-tested)
1. tz-aware vs naive datetime crash (adapters emit aware; tests were all-naive).
2. Zero-note sessions were re-billed on every refresh → durable version-aware
   distill checkpoint.
3. Meta-transcript content derailed JSON extraction → mark-as-zero-knowledge
   handling.
4. **84% of the raw corpus was harness exhaust** — evaluation transcripts
   stored as ordinary session files with no distinguishing metadata. Fixed
   with an ingest-level signature filter. Real corpus: **401 sessions across
   two tools**. Lesson: *corpus curation is a first-class problem in
   AI-session mining.*
5. Orphaned embeddings after corpus cleanup crashed link-building → tolerant
   rebuild.

## Free-tier economics (measured, then confirmed on the provider console)
Per-model daily token budgets are independent — chaining models as each
budget exhausts (with resumable checkpoints) processed the full corpus for $0.
Observed ~1.2–2k tokens/session at a 16k-char render cap.

## Degradation observed in production
Sustained 429s → per-session skip + clean resumable abort; exhausted model at
lens time → results render without LLM garnish; label/status failures →
cached/unknown fallbacks.
